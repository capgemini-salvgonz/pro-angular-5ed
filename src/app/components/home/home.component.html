CONCEPTOS BÁSICOS

Estructura simple de un proyecto Angular

La siguiente imagen describe los archivos y carpetas que se agregan a un nuevo proyecto mediante el comando ng new y que proporcionan el punto de partida para la mayoría de los desarrollos en Angular.

En esta imagen se reflejan algunos de configuración utilizados en tiempo de desarrollo, herramientas, procesos y archivos de salida.

Fig. 1. Espacio de trabajo en un proyecto nuevo.

Tabla. 1. Descripción básica de archivos y directorios en un proyecto nuevo.

Archivo

Descripción
package.json

Este archivo contiene detalles de los paquetes NPM requeridos por la aplicación y las herramientas de desarrollo y define en la sección de scripts los comandos que ejecutan las herramientas de desarrollo (Angular CLI) como: ng build, ng serve, ng test.

tsconfig.json

Este archivo contiene los ajustes de configuración para el compilador de TypeScript. No se necesita cambiar la configuración del compilador en la mayoría de los proyectos de Angular.

tsconfig.app.json

Este archivo contiene opciones de configuración adicional para el compilador de TypeScript, como la ubicación de los archivos de código fuente, definición de tipo de archivos y la ubicación de salida del compilador.

angular.json

Este archivo contiene opciones de configuración adicional para el compilador de TypeScript, como la ubicación de los archivos de código fuente, definición de tipo de archivos y la ubicación de salida del compilador.

node_modules

Este directorio contiene los paquetes NPM que serán requeridos por la aplicación y por las herramientas de desarrollo de Angular.
