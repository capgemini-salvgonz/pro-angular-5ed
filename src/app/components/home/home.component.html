Tabla. 1. Descripción básica de archivos y directorios en un proyecto nuevo.

Archivo

Descripción
package.json

Este archivo contiene detalles de los paquetes NPM requeridos por la aplicación y las herramientas de desarrollo y define en la sección de scripts los comandos que ejecutan las herramientas de desarrollo (Angular CLI) como: ng build, ng serve, ng test.

tsconfig.json

Este archivo contiene los ajustes de configuración para el compilador de TypeScript. No se necesita cambiar la configuración del compilador en la mayoría de los proyectos de Angular.

tsconfig.app.json

Este archivo contiene opciones de configuración adicional para el compilador de TypeScript, como la ubicación de los archivos de código fuente, definición de tipo de archivos y la ubicación de salida del compilador.

angular.json

Este archivo contiene opciones de configuración adicional para el compilador de TypeScript, como la ubicación de los archivos de código fuente, definición de tipo de archivos y la ubicación de salida del compilador.

node_modules

Este directorio contiene los paquetes NPM que serán requeridos por la aplicación y por las herramientas de desarrollo de Angular.

Proceso de construcción

  • Cuando se ejecuta el comando ng serve, el proyecto se crea para que pueda ser utilizado por el navegador. Este es un proceso que requiere tres herramientas importantes: el compilador de TypeScript, el compilador de Angular y un paquete llamado Webpack.

  • Las aplicaciones de Angular se crean utilizando archivos TypeScript y plantillas HTML que contienen expresiones, ninguno de los cuales puede ser entendido por los navegadores. El compilador de TypeScript es responsable de compilar los archivos TypeScript en JavaScript, y el compilador Angular es responsable de transformar las plantillas en sentencias de JavaScript que usan las API del navegador para crear los elementos HTML en el archivo de plantilla y evaluar las expresiones que contienen.

  • El proceso de compilación se administra a través de Webpack, que es un package module, lo que significa que toma la salida compilada y la consolida en un módulo que se puede enviar al navegador. Este proceso es conocido como empaquetado, que es una descripción bastante discreta para una función tan importante, y es una de las herramientas clave en la que se confiará mientras se desarrolla con Angular, no tenemos que profundizar ni ocuparnos de esta ya que, es administrada para nosotros por las herramientas de desarrollo de Angular.

  • Cuando ejecuta el comando ng serve, se puede ver una serie de mensajes a medida que Webpack procesa la solicitud. Webpack comienza con el código en el archivo main.ts, que es el punto de entrada para la aplicación (y que se especifica en el archivo angular.json ) y sigue las declaraciones de importación que contiene para descubrir sus dependencias, repitiendo este proceso para cada archivo en el que hay una dependencia. Webpack se abre camino a través de las declaraciones de importación, compilando cada TypeScript y archivo de plantilla en el que se declara una dependencia para producir código JavaScript para toda solicitud.

  • En el ejemplo mostrado en la Fig. 1. Webpack carga el archivo declarado en la propiedad "build.options.main":"src/main.ts" que se encuentra dentro de la sección architect del archivo angular.json, el cuál contiene la dependencia de un root-module definido en el archivo de código fuente app.module.ts y este a su vez define un componente que se encuentra en el archivo app.main.ts, de esta manera Webpack hace todo el recorrido para poder compilar en cadena los fuentes necesarios y finalmente poder usar el selector llamado app-main del archivo app.main.ts dentro del archivo index.html

Application Bundle

El proceso de compilación completo se realiza solo cuando se ejecuta por primera vez el comando ng serve, posteriormente, los paquetes se reconstruyen si alguno de los archivos que componen a la aplicación cambian.

Tabla. 2. Bundles producidos por el proceso de construcción de Angular.

Archivo

Descripción
main.js

Este archivo contiene la salida de la compilación del directorio src/app

polyfill.js

Este archivo contiene los Polyfills de JavaScript (fragmento de código que se utiliza para proporcionar una funcionalidad moderna en navegadores antiguos que no lo admiten de forma nativa) requeridos por las características usada en la aplicación y que no son soportadas por el browser target.

runtime.js

Este archivo contiene el código encargado de cargar otros módulos.

style.js

Este archivo contiene código JavaScript que agrega las hojas de estilo CSS globales de la aplicación.

vendor.js

Este archivo contiene los paquetes de terceros de los que depende la aplicación, incluyendo los paquetes de Angular.
